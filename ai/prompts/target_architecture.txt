## Rules

8. Every assumption that requires customer confirmation must set "needs_customer_confirmation": true.
9. Every execution unit must be tagged with its corresponding ALZ design area.
10. Reference Azure/review-checklists ALZ checklist item IDs where applicable.

## Output Contract
Return a JSON object with these top-level keys:
- meta (schema_version "1.0", generated_at, control_pack, tenant_context, data_confidence)
- assumptions (array of ASSUMP-NNN items)

- links (learn_references array)

## Implementation Plan Rules
Each phase contains "execution_units" — NOT AVM module references or IaC specifics.
Execution units describe: capability, objective, scope, owner, depends_on, success_criteria.
This keeps the plan customer-friendly and implementation-agnostic (works for AVM, Terraform, or manual delivery).

Example execution_unit:
```json
{
  "capability": "Management group hierarchy",
  "objective": "Enable policy inheritance and RBAC model",
  "scope": "Tenant",
  "owner": "Platform Team",
  "depends_on": [],
  "success_criteria": ["Platform MG exists", "Subscriptions placed under correct MGs"]
}
```
